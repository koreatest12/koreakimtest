# 프로그래밍 자기 개발을 위한 자기소개서 작성 워크플로우

## 개요
본 문서는 프로그래밍 분야에서 자기 개발을 강조하는 자기소개서를 작성하기 위한 단계별 워크플로우를 제시합니다. 각 단계는 실무 경험, 학습 계획, 성과 측정을 체계적으로 정리하여, 읽는 이에게 명확하고 설득력 있는 메시지를 전달하는 것을 목표로 합니다.

## 1. 준비 단계
1. **목표 독자 파악**
   - 지원 기업, 팀, 혹은 프로그램의 핵심 가치와 인재상 조사
   - 채용 공고, 조직의 기술 블로그, 공식 발표 자료를 통해 요구 역량 분석
2. **경험 목록화**
   - 프로젝트, 인턴십, 오픈소스 기여, 대회 등 주요 경험 타임라인 정리
   - 역할, 사용 기술, 기여도, 성과(정량/정성) 메모
3. **핵심 역량 정의**
   - 언어/프레임워크 숙련도, 문제 해결력, 협업 능력, 학습 민첩성 등 키워드 선정
   - 역량별로 뒷받침할 사례와 지표 연결

## 2. 구조 설계 단계
1. **전체 아웃라인 작성**
   - 도입부: 핵심 메시지(개발자로서의 지향점) 요약
   - 본문 1: 대표 경험 소개 (문제 → 행동 → 결과 순)
   - 본문 2: 학습/자기 개발 전략 및 실행 사례
   - 본문 3: 협업 및 커뮤니케이션 사례, 조직에 기여 가능한 부분
   - 결론: 향후 성장 계획과 조직과의 시너지 강조
2. **스토리텔링 요소 배치**
   - STAR(Situation, Task, Action, Result) 혹은 CAR(Context, Action, Result) 프레임워크 활용
   - 각 단락마다 핵심 메시지를 한 문장으로 정리
3. **자료와 지표 정리**
   - GitHub 리포지터리, 블로그 포스트, 발표 영상 등 참고 링크 수집
   - 코드 품질 지표, 배포 지표, 사용자 피드백 등 객관적 근거 확보

## 3. 작성 단계
1. **초안 작성**
   - 아웃라인에 따라 각 단락을 5~7문장 내외로 작성
   - 기술 용어는 필요한 경우만 사용하고, 비전문 독자도 이해할 수 있도록 설명 보완
2. **차별화 요소 강화**
   - 고유한 문제 해결 방식, 시행착오, 학습 전략 등을 구체적인 사례로 서술
   - 학습 커뮤니티 참여, 스터디 운영, 멘토링 경험 등 자기 개발 활동 포함
3. **톤 & 매너 조정**
   - 진정성 있는 1인칭 시점 유지
   - 성취를 과장하지 않되, 임팩트를 명확히 전달

## 4. 검토 및 피드백 단계
1. **자체 검토 체크리스트**
   - 문맥 흐름이 자연스러운가?
   - 경험과 역량이 명확히 연결되어 있는가?
   - 불필요한 중복, 장황한 표현이 없는가?
   - 오탈자, 맞춤법 오류가 없는가?
2. **피드백 수집**
   - 멘토, 동료, 커리어 컨설턴트에게 리뷰 요청
   - 내용, 구조, 톤에 대한 개선 제안 수렴 후 반영
3. **버전 관리**
   - 주요 수정 이력을 문서 하단 혹은 별도 CHANGELOG에 기록
   - 지원 기관별로 맞춤형 버전 분리 관리

## 5. 최종 제출 준비 단계
1. **포맷 정리**
   - 제출 플랫폼 요구 형식(문서 템플릿, 글자 수, 파일 형식) 확인 후 맞춤 조정
   - PDF 제출 시 폰트, 레이아웃, 링크 동작 여부 재확인
2. **부속 자료 정리**
   - 이력서, 포트폴리오, 추천서 등 부속 문서와 메시지 일관성 유지
   - 제출용 버전은 클라우드 및 로컬에 이중 백업
3. **최종 리뷰 및 제출**
   - 제출 마감일과 시간 재확인
   - 제출 확인증 또는 이메일 수신 여부 체크

## 부록: 추천 템플릿 구조
```markdown
# 자기소개서 제목 (지원 포지션 명시)

## 1. 소개 및 비전
- 현재 역할 및 주요 관심 기술
- 개발자로서의 목표와 가치관

## 2. 핵심 프로젝트 경험
- 프로젝트명 / 기간
- 상황 및 도전 과제
- 수행한 역할과 사용 기술
- 정량/정성 성과 및 배운 점

## 3. 자기 개발 및 학습 활동
- 최근 학습한 기술 혹은 자격증 준비 현황
- 스터디, 커뮤니티, 오픈소스 기여 사례
- 학습 결과를 실무에 적용한 사례

## 4. 협업 및 커뮤니케이션
- 팀 내 협업 경험과 역할 분담
- 갈등 조정, 코드 리뷰, 문서화 등 협업 역량 사례

## 5. 향후 성장 계획
- 입사 후 기여하고 싶은 영역
- 중장기적인 커리어 목표와 실행 계획

## 부록 (선택)
- 포트폴리오 링크, GitHub, 블로그, 발표 자료 등
```

## CLI 도구로 워크플로우 실행하기

> 🛠️ `python -m src.cli.cli_main selfintro <command>` 형식으로 실행합니다.

| 명령 | 설명 |
| --- | --- |
| `list` | 전체 워크플로우를 체크리스트 형태로 출력합니다. |
| `run [--non-interactive]` | 각 단계를 순차적으로 안내합니다. 기본적으로 Enter 키로 다음 항목으로 이동하며, `--non-interactive` 옵션으로 로그 형태 출력이 가능합니다. |
| `generate-template [--output 파일]` | 추천 마크다운 템플릿을 생성합니다. 기본 파일명은 `self_intro_template.md` 입니다. |

```bash
# 예시: 워크플로우 진행
python -m src.cli.cli_main selfintro run

# 예시: 템플릿 생성
python -m src.cli.cli_main selfintro generate-template --output my_intro.md
```

> ℹ️ 본 워크플로우는 지원 기관의 제출 양식 및 요구사항에 따라 유연하게 조정할 수 있으며, 최신 채용 동향을 수시로 확인하는 것을 권장합니다.
